Copy, Paste and Move Elements

From the Data Modeler Navigator, copy, paste and move tables actions can be performed. These actions are detailed below:

•      Copy: To access this option, go to the table that you want to copy, right click and select Copy. When a single table is copied, the FKs will not be copied. If several interrelated tables are copied, the FKs with the relationship between these tables will be copied.

 

 

•     Paste: This action will always be enabled provided that:

-     There is some content in the clipboard.

-     There is only one target element selected in the navigator.

-     The element selected in the navigator is a schema.

This action also involves pasting the elements linked underneath that were copied along with the target element. When just one table is selected for copying, the FKs will not be pasted.

 

 

If the element to be pasted already exists in the target, a window is displayed that can be used to change its name.

 

 

•      Move: It will be enabled when several schemas are available in the project. If only one schema were available, it would not be possible to move the table from its current location. To access this option, go to the table or tables that you want to move, right click and select Move.

 

 

A window is displayed in which the target schema where the table will be moved must be selected. Once this is completed, click .

 

 

If the element to be moved already exists in the target, a window appears for changing its name.

 

 

Finally, the renamed table will be displayed in the Data Modeler Navigator.

 

 

The copy and move actions are also available in the Modeler Diagram. They can be located by right clicking within the Edit option. The paste action will only be available when a schema is selected in the Data Modeler Navigator.
