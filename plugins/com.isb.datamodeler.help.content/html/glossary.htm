Glossary

•      Catalogues: Collection of schemas (and other descriptors).

•      Columns: Component of a table with a name, data type, default value and the option of being or not being null.

•      Database Environment: Specific implementation of a manufacturer of a collection of catalogues, authorisation IDs, etc.

•      Foreign Key: A foreign key is a reference to a key in another table. It determines or establishes the relationship that exists in two tables. Foreign keys do not need to be unique keys in the table where they are located; however, they do need to be unique in the table where they are referenced. For example, the department code can be a foreign key in the Employees table. Obviously, several employees are allowed in the same department, but only one department will exist.

•      Primary Key: A primary key is a unique key chosen among all the candidates to uniquely define all the other table attributes for specifying the data that will be related to the other tables. This is performed by using foreign keys. There can only be one primary key per table and no fields in that key can contain NULL values.

•      Relational Data Model: This is the model that can be used to establish interconnections (relationships) between the data (that are saved in tables). These connections are used for relate the data from both tables.

•      Schemas: A schema is a collection of descriptors (tables, views, sequences, routines, etc.)

•      Table: Set of records (rows and columns).

•      Unique Key: Each table can contain one or more fields whose values uniquely identify each record in that table. That is, there cannot be two or more different records whose values in those fields are identical. This set of fields is called unique key. There can be several unique keys in a specific table, and each of them is usually call a candidate primary key.
