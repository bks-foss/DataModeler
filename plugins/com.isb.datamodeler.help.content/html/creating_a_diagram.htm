Creating a Diagram

The relational data model is created in the diagram. The relationships between these model elements are established in the diagram. The diagram belongs to a schema.

 

Steps

1.     To create a diagram, in the Data Modeler view, right click the schema and select New>Diagram.

 

 

2.  A wizard appears that guides you through the diagram creation process. The Wizard window contains the following fields:

-     Project: Enter the name of the project associated with the application.  The default will be to display the name of the project where you are currently positioned. To select an existing project, click  .

-     Schema: Enter the name of the schema associated with the project.  The default will be to display the name of the schema where you are currently positioned. To select an existing schema, click  .

-     Name: The automatically generated name, which consists of the prefix 'DD_' plus the name of the acronym and the name of the schema, will be displayed. A name can be added manually.

-      Description: Include a description.

      Click  to end the process.

 

 

3.   The Data Modeler view displays the new diagram you have just created.

 

 

4.    The diagram editor appears. It contains a palette with the elements required for the actually modelling.

See: Diagram Editor Palette

 

 

5.   From the Data Modeler view, at the diagram level, right click and a contextual menu appears displaying the following actions:

-      Delete from Model: To delete the diagram.

-      Deselect All: To deselect the project(s) currently active.

-      Lastly, the icons for other existing modelling projects will be displayed in the workspace. They can be enabled.

 

 

6.   To add an element from the palette, select it by clicking and, then, click the position on the editor where you want to release it. To find out which elements can be added to the diagram, see: Diagram Editor Palette

 

 

7.    The Data Modeler view displays the new element you have just added.

 

 

With your cursor positioned over the table, if you right click, a popup window appears displaying the following actions (among others):  

-     New>Column: A column is a component of a table to which a name, a data type, a default value and an indicator defining whether null value can be assigned.

See: Creating a Column

-      New>Unique Key: To specify a group of table columns used for identifying the records uniquely. The columns form the restriction in an order and can have a null value.

See: Creating a Unique Key

-      Delete from Model: To delete the element from the diagram.

-    Detect Possible Incoming Relationships: This is used to detect, for each selected table, groups of columns that can be referenced from other tables.

See: Detect Possible Relationships

-     Detect Possible Outgoing Relationships: This is used to detect, for each selected table, groups of columns that can be referenced to keys of other tables (from both the application and external tables)

See: Detect Possible Relationships

-      Deselect All: To deselect the project(s) currently active from the view.

-      Lastly, the icons for other existing projects will appear in the workspace. They can be enabled.

 

 

8. From the diagram editor, with your cursor positioned over an element, if you right click, a popup window appears with the following actions, among others:

-     Remove from Diagram: To delete the element from the diagram, but not from the modelling. The elements will not be deleted from the model.

-     Remove from Model: To delete the element from the diagram and also from the actual modelling.

-     Format: Set of options that can be used for changing the visual aspect of the diagram elements, such as reordering, alignment, size, etc.

-     Filters: To apply filters for hiding or showing elements and compartments.

-   Restore Related Tables: This option will draw in the diagram the model tables (from the same schema) with which the chosen table has a relationship (outgoing) and which are not diagrammed.

-     Restore Relationship: This option draws in the diagram the model relationships from/to the chosen table to/from other tables of the diagram and which were not diagrammed.   The relationships that are restored can be incoming, outgoing and they also work with external tables.

If several tables are selected in the diagram and this option is selected, the action will be applied to all the selected tables.

If the action is launched from the diagram and nothing has been selected, it will be applied to all the tables in the diagram.

-     Restore Tables: This option restores in the diagram the model tables that were previously deleted in the diagram, and with which the table from which the restore action is being performed had an (outgoing) relationship, as well as its relationships.  

-      Show Properties View: To open the Properties view with information at the level of the element where you are currently positioned.

-      Detect Possible Incoming Relationships: This is used to detect possible incoming relationships from the model tables to an application table

See: Detect Possible Relationships

-     Detect Possible Outgoing Relationships: This is used to detect possible relationships between the tables that belong to the project application and other model tables.

 See: Detect Possible Relationships

 

 

 

 Depending on the element on which you are positioned, one of these actions can be disabled

9.   If you need to find a specific table in the diagram, position your cursor on the white background of the editor, right click and in the popup window select Find Table in the Diagram or, else, use Ctrl+Shift+D keys.

In the popup window displays the tables included in the diagram, select the table that you are searching and click  .

 

 

The selected table is highlighted in the diagram. If the table is not found in the open diagram (or there is no diagram opened), it will be displayed in the navigator.

9.   To set up the relationships between the tables added to the diagram, while positioned in the palette, click the relationship type to be created and, then, click the source element and before releasing it, click the target element.

 

 

To find out which relationships can be set up between the different elements included in the diagram, see: Diagram Relationships

 

In addition, when a diagram is included in the project, a set of buttons is displayed in the toolbar that can be used for the following:

•    Select: Click  and in the combo list, choose the option that best suits your needs.  

 

 

This combo list contains the following options:

-      Select All: All the elements and connectors available in the diagram are selected.

-      Select All Shapes: Only the elements that exist in the diagram are selected.

-      Select All Connectors: Only the relationships that exist in the diagram are selected.

•     Arrange: Click  in this group of buttons, and in the combo list, select whether all of the editor components should be arranged (Arrange All () or only those that are selected (Arrange Selection).

 

 

•        Reduced Column Decorator: Only the name is displayed for each column.

•        Extended Column Decorator: The name and attributes are displayed for each column.

•        All Connector Labels: If this button is selected, all of the attributes of the relationship are displayed in the editor.

•        No Connector Labels: If this button is selected, all of the attributes of the relationship are hidden in the editor.

•      Show Compartments: To hide/show the information available in the table compartments.
