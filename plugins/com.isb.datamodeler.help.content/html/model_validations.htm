Model Validations

Data Modeler provides, on one hand, automatic validation, which checks maximum length and invalid characters in names and descriptors of elements.

On the other hand, it also provides manual validation, which helps to detect errors in the model. Basic validations of the model and the guidelines are performed. A check will be run to ensure the model is correct and to report any errors that may occur.

 

Steps

 

1.   From the Data Modeler Navigator, right click the project where the validation needs to be run and then select Validate Model.

 

 

2.   Basic validations of the model and the guidelines are performed. Two different scenarios may occur:

-      No errors exist in the model: A window will appear to report this fact.

 

 

-      Errors exist in the model: A validation report is displayed in which general validations are distinguished from specific validations for models with databases. The messages displayed are grouped by validation type (basic validation and guideline validation) and within each type errors will first be displayed and, then, warnings. The messages displayed for models with databases include the literal "[Data Dictionary]" ("[Diccionario de Datos]") as prefix.

The messages displayed may be selected (using the CTRL key) and may also be copied by right clicking and selecting Copy  from the popup menu or using the CTRL+C keys.

 

 

This information can also be exported to an .html file so that it can be reviewed afterwards and in order to more easily send the results to the interested party. In order to do this, click , located at the top, right-hand side of the view and then specify the location and the file name in the popup window.
