Detect Possible Relationships

Purpose

Data Modeler can be used to detect possible relationships between the tables that belong to the project application and other model tables.

 

 

Steps

1.    In the Data Modeler Navigator or Diagram Editor, position your cursor over the table(s) from which possible relationships must be detected. Right click and in the popup menu, select one of the following options:

-     Detect Possible Incoming Relationships: Data Modeler will detect for each selected table, groups of columns that can be referenced from other tables.

-     Detect Possible Outgoing Relationships: The Data Modeler will detect for each selected table, groups of columns that can be referenced to keys from other tables (from both the application and external tables)

 

 

2.    A window appears displaying the columns that can be references to keys in other application tables and external tables. The columns displayed must fulfil the following:

-     Its name matches or starts with the column name of the possible referenced key.

-     Its type and parameters match those in the column of the possible referenced key.

-     At least one of the columns must not already belong to a FK.

 

The information displayed for each table of the schema that contains possible detected relationships is as follows:

-      Child Table / Suggested Columns: Suggested columns of the child table for the relationship.

-      Parent Table. Key Columns: Name of the parent table and Columns making up the suggested unique key.

-      Select: Checkbox used for selecting the relationships that must be established.

Click .

 

 

The selected relationships will appear in their corresponding table within the modeler diagram. If the action was executed from the diagram and the tables that take part in the relationship are included in the diagram, the relationship will be automatically drawn. Otherwise, to view one of the relationships of a table, position your cursor over the desired table, right click and select the Restore Links option. The relationships are added to the table as FKs (Foreign Keys). Repeat this action for each table whose relationships you wish to display
