Creating a Column

Purpose

A column is a component of a table to which a name, data type and default value are assigned. Depending on the data type assigned to the column, more or fewer attributes to be completed will be displayed.

Columns are defined in the tables.

 

 

Steps

1.    Several options are available for including a column in a table:

-      In the Data Modeler Navigator view, right click the table where the column must be included and in the popup menu, select New>Column

-      In the editor palette, click the element  , go to the table where you want to create the column and click again.

The column appears in the table.

 

 

2.   The Properties view displays the following fields that are used for parameterising the column:

-      Cancellable: Boolean value indicating whether a null value (False) or not (True) can be included.

-     Primary Key: Boolean value indicating whether the column represents the primary key of the table.  These columns cannot be cancellable, so if their value is True, the value for the Cancellable field will also be True.

-      Description: Column description.

-      Name: Name identifying the column

-     Part of Foreign Key: Indicates whether the column is a reference to a key in another table. Determines or establishes the relationship between two tables.

-      Primitive Type: Combo list where the data types predefined for each database can be selected

-      Default Value: Default value
