Import Tables

Purpose

Data Modeler can be used to import tables from the repository of the data dictionary.

 

 

Steps

1.    In the Data Modeler Navigator, go to the modelling project where the tables will be imported, right click and, in the popup menu, select the Import Tables option.

 

 

2.    A wizard appears to guide you through the table import process. At least one of the following fields must be completed:

-     SubApplication: Type the subapplication code from which to import the tables. The default value of this field is the subapplication code of the selected modelling project.

-     Entity Name: Name of the table(s) to be imported. ‘*’ can be used as wildcard for n characters or ‘_’ for just 1 character.  At least 3 characters (apart from * or _) need to be added in order for the search to be valid.

Once the validations performed by the tool are fulfilled, the   button will be enabled.

 

 

The wizard window displays a list of the tables that fulfil the search requirements and which have been located in the data dictionary repository.  The information displayed for each table is the following:

-     Name: Corresponds to the name of the table.

-     Modelled: Indicates whether or not the entity is in our model.

-     CheckOut: Indicates whether the entity has been checked out in the dictionary.

-     Subapplication: Subapplication to which the table belongs.

-     Schema: Displays the name of the schema where the table is located. If the dictionary table contains information on the schema to which it belongs, this information will be displayed. If the table is included in any of the schemas of the model subapplication, the information on that schema will be displayed.

Select the entities to be imported by using the checkbox located next to the name. Entities from different subapplications cannot be selected for being imported at the same time.

The window displays the Default Schema field to add the name of the schema where the tables will be downloaded. This field will be used if no value appears in the Schema column, using the value of this field for the schema.

Click .

 

 

The wizard displays the Model Merge window split in two sections:

-     The top section contains the differences between the repository and the local project.

-    The lower section is split up into two parts. The right part contains the tables (and all of its associated elements, except for information on FK restrictions) that were selected to be imported and the left part contains all the tables from the model.

 

 

In addition, each of the parts contains the Differences and Properties tabs where, depending on the currently active tab, different information can be viewed.

Once the wizard is finished, the imported tables will appear within the modelling project. Those from different subapplications will appear with their name between the greater than and less than symbols (<<  >>).
