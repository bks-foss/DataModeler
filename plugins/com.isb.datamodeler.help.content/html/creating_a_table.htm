Creating a Table

A table is an ordered collection of one or more columns in an unordered collection of records. Tables are defined in the schemas.

 

 

Steps

1.    Several options are available for including a table in a schema:

-      In the Data Modeler view, right click the schema where the table must be included and in the popup menu, select New>Table. The table appears in the navigator. To include it in the diagram, select it and drag it to the editor.

-      In the editor palette, click the element , go to the location in the editor where you want to create the table and click again.

 

 

2.  The Properties view displays the following fields that are used for parameterising the table:

-     Columns: Shows the table columns separated by a comma. The name, type and parameters are displayed for each column. If you click , a windows appears where the column order can be changed.

 

 

-     Description: Table description.

-     Schema: Name of the schema to which the table belongs. This field cannot be edited.

-     Name: Corresponds to the name of the table.

-     Referencing Foreign Keys:  Represents the foreign keys (FK) that reference this table.

-     Restrictions: Shows a list with the restrictions, separated by a comma.

 

 

A table may contain the following elements:

-     Columns: Creating a Column

-      Unique Keys: Creating a Unique Key
