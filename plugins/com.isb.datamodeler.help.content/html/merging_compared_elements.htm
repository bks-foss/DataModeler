Merging Compared Elements

When two modelling projects are functionally compared, the lower part of the Editor displays the differences that exist between both. The differences that appear highlighted are the changes between the source element and the target element on which a merge can be performed.

Depending on the type of change viewed, the Comparison editor includes the following buttons partially or totally enabled allowing performing the following merge types:

•       Copies all the content of the element displayed in the left section to the content of the element in the right section, making the content of both elements to be identical.

•       Copies all the content of the element displayed in the right section to the content of the element in the left section in order to make the content of both elements to be identical. Please note the contents that appear in the left section would be lost.

•       Copies the change highlighted in the left section to the fragment highlighted in the right section. This action overwrites the fragment highlighted in the right section.

•       Copies the change highlighted in the right section to the fragment highlighted in the left section. This action overwrites the fragment highlighted in the left section.

 

In addition, depending on the type of resource compared, the following buttons will be displayed partially or totally so that you can scroll through the differences found:

•       Highlights the next difference found in the compared elements.

•       Highlights the previous difference found in the compared elements.
