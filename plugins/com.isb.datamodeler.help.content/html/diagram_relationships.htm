Diagram Relationships

Relationships determine how data between two entities are related.

The selection of relationship types is essential for the built-in referential integrity of the data models.

The following relationships can be established between tables:

•      : It is represented by a solid line. The cardinality and the corresponding role are displayed at the end of the relationship.  Identificatory relationships transmit the primary key of the parent entity to the primary key of the child entity.

 

 

When this type of relationship is established in the Properties view, the following fields are displayed for configuration

 

 

-   Base Cardinality: The source (child table) can contain the following values:

• 0…1

• 0…*

• 1…*

• 1

-    Base Role: represents the role played by the child table in the relationship. For example, in a relationship between the Employee table (child) and the Company table (parent), it could be 'works for'.

-     Base Table: name of the table from which the relationship originates.

-     Description: To include a brief description on the relationship.

-     Identificatory: Boolean value indicating whether the relationship is identificatory (true) or not identificatory (False).

-     Members: Columns in the child table that are part of the restriction. If columns with an Identificatory relationship are included, this relationship becomes not identificatory (ID value = False).

-     Name: Name of the relationship. Data Modeler indicates a default value that cannot be modified.

-     Parent Cardinality: only the value of 1 is possible in this case.

-     Parent Role: represents the role played by the parent table in the relationship. For example, in a relationship between the Employee table (child) and the Company table (parent), it could be 'employs'.

-     Parent Table: name of the target relationship table.  

-     Referenced Members: columns making up the unique restriction.

-     Unique Key: the values of these fields uniquely identify each record in the table (UK).

-     Unique Restriction: unique restriction (PK or UK) belonging to the parent table to which the child table is referencing through the FK.

-     When Deleted: Click  and in the combo list, select the referential integrity action to be used when a row is updated in the parent table.

 

 

-     When Updated: Click   and in the combo list, select the referential integrity action to be used when a row is updated in the parent table.

When a new relationship is created, a wizard will be displayed if:

•      the member columns already exist in the child table even though their names do not match.

•      the parent table includes several keys

 

The name of the member columns will be displayed for each key so you can define the list of member columns for the relationship by choosing among the columns of the child table with the corresponding type or by adding a new column using the columns from the referenced key.

 

 

This wizard displays the columns of the child table on the left and the columns of the referenced key on the right. The Members section appears in the middle section and contains one member column per FK-referenced members. Columns from the list of referenced members and from the child table can be included in this section.

 

This wizard runs a series of validations in order to add the columns to the Members section. These validations are as follows:

•      The number of member columns must match the number of referenced columns.

•      The type of each member column must match the corresponding referenced column.

•      The name of the new member column must be different from another new column or an existing one in the table.

 

The Members section can be completed in different ways:

•      If the column to be added as a member has already been created on the child table, this column can be used instead of including a new one. To do so, select the column from the list that appears on the left and click  . The column will be displayed in the Members section.

 

 

•     To add the column using the list appearing on the right and containing elements referenced from the foreign key, then, select the column and click   or   to copy all the columns. In this case, the column not only would be added to the Members section, but it would also be created in the list on the child table.

 

 

The icon   will appear next to the newly added column in order to differentiate it from existing columns on the child table.

Whenever a new column is added by using a referenced column, a check will b run to verify whether there is already a column with the same name on the child table by using, whenever necessary, a numeric suffix to prevent name conflicts from occurring.

 

 

Add as many columns as necessary in order to ensure there is an identical number of member columns and of members referenced from the Foreign Key.

 

If the order of the members and the order of the referenced members are not the same, use  and  to modify the order accordingly; otherwise, the    button will not be enabled.

Click   to finish the process.
