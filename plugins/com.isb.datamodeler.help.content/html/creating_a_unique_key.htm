Creating a Unique Key

Each table can contain one or more fields whose values uniquely identify each record in that table (UK). That is, there cannot be two or more different records whose values in those fields are identical. There can be several unique keys in a table.

 

 

Steps

 

1.   Several options are available for including a Unique Key in a table:

- In the Data Modeler Navigator view, positioned on the table where the column must be created, right click and in the popup menu, select New>Unique Key.

- In the editor palette, click the element  , go to the table where you want to create the unique key and click again.

The unique key appears in the table.

 

 

The Properties view displays the following fields used for parameterising the unique key:

-      Foreign Key: Indicates whether it is a reference to a key in another table. Determines or establishes the relationship between two tables.

-      Description: Description of the unique key.

-      Members: To include the columns that will be part of the key. Click the button on this field  and a window appears where the columns of the key can added, deleted and sorted.

 

 

-     Name: Name of the unique restriction.

-     Base Table: Table where the unique key is included.
