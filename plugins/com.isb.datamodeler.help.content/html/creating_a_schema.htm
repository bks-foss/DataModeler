Creating a Schema

A schema is a collection of descriptors (tables, views, sequences, routines, etc.) with the different objects of the model. Each schema must belong to an application and subapplication.

 

Steps

1.   To create a new schema, in the Data Modeler Navigator view, right click and select New>Schema.

 

 

2.   A wizard appears to guide you through the schema creation process. The Wizard window contains the following fields:

-    Project: Enter the name of the project associated with the application.  The default will be to display the name of the project where you are currently positioned. To select an existing project, click  .

-      Name: Type the name to be assigned to the schema. Two schemas with the same name cannot exist in the same model.

-      Description: Include a description.

-      Delete from Model: To delete the modelling schema.

-      Deselect All: To deselect the project(s) currently active from the view.

-    Lastly, the existing projects will be displayed in the workspace, where they can be selected. Marking or unmarking the project will mean it is either included or not included in the view. In addition, if a modelling project is deselected, any associated editors that may be open will be closed.
