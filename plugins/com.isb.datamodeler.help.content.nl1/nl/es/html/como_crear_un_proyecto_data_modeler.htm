Cómo crear un Proyecto Data Modeler

El proyecto Data Modeler contendrá los esquemas, tablas y todos aquellos elementos necesarios para definir el modelo de datos.

 

Pasos

 

1.    Para crear un nuevo proyecto, dentro de la vista Navegador Data Modeler, haga clic en el botón derecho, y seleccione Nuevo>Proyecto Data Modeler.

 

 

2.    Aparece un asistente que le guía durante la creación del proyecto. Contiene los siguientes campos:

-    Acrónimo de la Aplicación: Cadena que identifica la Aplicación. Debe contener 6 caracteres.

-    Código de Subaplicación: Código numérico que identifica la subaplicación.

-    Nombre el proyecto: Se completa de forma automática a partir del acrónimo, añadiéndole el sufijo “_MD”.

-    Tipo de Base de Datos: Actualmente el único tipo de Base de Datos disponible es Nuevo Diccionario de Datos

-    Descripción: Incluya una descripción del proyecto.

Por último, haga clic en el botón .

 

 

3.    En la vista Navegador Data Modeler se muestra el proyecto que acaba de crear.

 

 

4.   Una vez creado el proyecto, en la vista Data Modeler sitúese en el proyecto y haga clic en el botón derecho del ratón. Aparece un menú contextual con las siguientes acciones:

-       Nuevo>Proyecto Data Modeler: Permite crear un nuevo proyecto de modelado.

-       Nuevo>Esquema: Permite crear un nuevo esquema.

Consulte: Cómo crear un Esquema

-      Eliminar del modelo: Permite eliminar del modelo el proyecto activo.

-      Validar Modelo: Permite realizar una validación que ayudará a detectar incorrecciones en el modelado.

Consulte: Validaciones del Modelo

-      Comparar> Con otro proyecto CVS: permite comparar una proyecto en local con otro que esté en el CVS.

Consulte: Comparación de local a CVS

-      Comparar>Con otro proyecto Local: permite comparar dos proyectos de modelado seleccionados en local

Consulte: Comparación de local a local

-      Deseleccionar todo: Permite deseleccionar de la vista el/los proyecto/s de modelado activos en ese momento.

-      Por último, se mostrarán los proyectos existentes en el área de trabajo y que pueden ser seleccionados. Marcar o desmarcar el proyecto supondrá incluirlo o no en la vista. Además si desmarca un proyecto de modelado se cerrarán los editores asociados que estén abiertos.
